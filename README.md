# EventMind

EventMind — прототип системы агрегации IT-событий и персонализированных рекомендаций на основе модели пользователя.

## Идея проекта

Система предназначена для:
- сбора информации об IT-событиях,
- хранения пользовательских предпочтений,
- формирования персональных рекомендаций,
- фиксации обратной связи пользователя.

В качестве пользовательского интерфейса используется Telegram-бот.  
Backend реализован в виде REST API на FastAPI.  
Для хранения данных используется SQLite.

## Архитектура

Система состоит из следующих модулей:

- `app/bot` — Telegram-бот
- `app/api` — backend API
- `app/db` — модели данных и подключение к БД
- `app/recommender` — модуль рекомендаций
- `app/ingestion` — загрузка и нормализация событий
- `data/events.json` — тестовый набор событий

## Реализованный функционал MVP

- настройка профиля пользователя через Telegram-бота;
- выбор интересующих тем;
- выбор формата и города;
- регистрация пользователя через API;
- загрузка событий из JSON;
- получение списка событий;
- получение персональных рекомендаций;
- запись пользовательских действий:
  - интересно;
  - не интересно;
  - сохранить.
- просмотр сохраненных событий через `/saved`;
- повторная настройка профиля через `/edit`;
- обновление модели пользователя на основе обратной связи;

## Основные команды Telegram-бота

- `/start` — первичная настройка профиля
- `/recommend` — показать рекомендации
- `/profile` — показать профиль и веса интересов
- `/saved` — показать сохраненные события
- `/edit` — изменить профиль

## Логика рекомендаций

На текущем этапе используется rule-based подход.

Итоговый score события формируется по:
- совпадению тематики;
- совпадению формата;
- совпадению города.

В дальнейшем возможно расширение рекомендательного модуля:
- анализом текстов событий с помощью NLP;
- векторным представлением события и профиля пользователя;
- динамическим обновлением модели пользователя на основе истории взаимодействий.
